OneBranch DevOps API

This API enables automation of CI/CD workflows including build execution, deployment management, and log retrieval. This documentation sample reflects real-world DevOps API documentation practices.

Base URL

https://api.onebranch.dev/v1

Protocol: HTTPS

Format: JSON

Authentication

Authorization: Bearer <access_token>

Trigger Pipeline

POST /pipelines/trigger

Request

{
  "repo": "user-service",
  "branch": "main",
  "triggered_by": "vinod"
}

cURL

curl -X POST https://api.onebranch.dev/v1/pipelines/trigger \
-H "Authorization: Bearer <token>" \
-H "Content-Type: application/json" \
-d '{
  "repo": "user-service",
  "branch": "main",
  "triggered_by": "vinod"
}'

Response

{
  "pipeline_id": "pl_7890",
  "status": "started",
  "timestamp": "2026-01-10T10:00:00Z"
}

Get Build Status

GET /builds/{id}

Response

{
  "build_id": "b_4567",
  "status": "success",
  "duration": "5m 32s",
  "artifact_url": "https://storage.onebranch.dev/builds/b_4567.zip"
}

Deploy Application

POST /deployments

Request

{
  "build_id": "b_4567",
  "environment": "production",
  "deployed_by": "vinod"
}

Response

{
  "deployment_id": "d_3321",
  "status": "in_progress"
}

Fetch Logs

GET /logs/{pipeline_id}

Response

{
  "logs": [
    "Cloning repository...",
    "Running tests...",
    "Build successful",
    "Deployment started"
  ]
}

Error Handling

CodeDescription
400Bad request
401Unauthorized
500Server error
